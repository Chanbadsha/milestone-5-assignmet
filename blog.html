Important FAQ

What is a closure in JavaScript?

A closure is a feature in JavaScript where an inner function has access to variables from its outer (enclosing) function, even after the outer function has returned. This allows the inner function to remember and continue to access the scope in which it was created.

What is the purpose of async and await in JavaScript?

async and await are used to work with promises in a more readable, synchronous-like manner. The async keyword makes a function return a promise, and await pauses the execution of the function until the promise is resolved or rejected, allowing for asynchronous code to be written in a way that looks synchronous.

How does event propagation work in JavaScript?

Event propagation refers to the way events flow through the DOM in three phases: the capturing phase (from the root to the target element), the target phase (when the event reaches the target element), and the bubbling phase (from the target element back up to the root). This allows event listeners to be set either during capturing or bubbling.

What is hoisting in JavaScript?

Hoisting is JavaScript's default behavior of moving variable and function declarations to the top of their containing scope before code execution. This means that variables and functions can be used before they are declared, although the value of variables declared with var will be undefined until assigned.

What are JavaScript arrow functions and how are they different from regular functions?

Arrow functions are a concise syntax for writing functions in JavaScript. Unlike regular functions, they do not have their own this, arguments, or new.target bindings, making them more suited for scenarios where functions need to inherit the this value from their surrounding context. They are often used in callback functions and functional programming.
